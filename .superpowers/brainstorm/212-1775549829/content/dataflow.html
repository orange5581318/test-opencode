数据流 & 核心机制

登录鉴权、权限控制、Mock 数据的工作方式

登录鉴权流程

  • 登录 → 获取 token → 存入 localStorage + Pinia userStore
  • Axios 请求拦截器自动带上 Authorization header
  • 响应拦截器统一处理 401（token 过期跳登录页）
  • 路由守卫 beforeEach：无 token 跳 /login，有 token 放行

动态路由 + 权限

  • 登录成功后请求用户权限菜单接口
  • 根据返回的菜单列表动态 addRoute() 注册路由
  • 侧边栏菜单由 Pinia permissionStore 驱动渲染
  • v-permission 指令：检查按钮权限码，无权限则 v-if 隐藏

Mock 数据方案

  • 使用 vite-plugin-mock，开发环境自动拦截 /api/* 请求
  • 每个模块对应一个 mock 文件（product.ts / order.ts 等）
  • mockjs 生成随机数据，支持分页、搜索参数模拟
  • 生产构建时 mock 自动关闭，无需改代码
